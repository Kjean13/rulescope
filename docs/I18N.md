# RuleScope i18n model

RuleScope uses **English as the canonical engine language**.

## Design

- analyzers and the scoring engine keep stable finding codes such as `NOISE-001` or `META-005`
- English remains the source of truth for programmatic outputs and internal logic
- human-facing renderers translate findings and recommendations at display time

## Behaviour

- console and CLI messages follow the selected language
- analyzer findings are translated in human-facing CLI and console output
- HTML and Markdown reports localize findings and recommendations
- JSON and SARIF remain canonical English for machine consumption

## Why this design

This avoids spreading `if locale == ...` conditions across analyzers and keeps the engine easier to test, easier to evolve, and consistent across all interfaces.
